## Key Decisions (v1.1)

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Sealed `EvmSignature` union | `sealed class` in Dart 3 | Compile-time prevention of Pitfall 1 (raw r‖s fed to passkey path) |
| Single-flight in abstract layer | `_pending` Future in `EvmSigner` | Pitfall 7 prevention at the correct abstraction level; passkey adapters don't need to worry |
| Injected passkey adapter | `Function` type parameter | SDK has no platform dependency; callers own the platform layer |
| Per-chain `BarzDeployment` registry | Immutable value object + `BarzDeployments` class | Pitfall 9 prevention; single canonical source for factory addresses |
| CREATE2 round-trip mandatory | Always verify via `TWEthereumEip1014` | Pitfall 2 prevention; no "trust but don't verify" code path |
| v0.6/v0.7 named constructors | Separate constructors → separate proto types | Pitfall 4 prevention; wrong version is a compile-time selection error |
| `attachSignature` as sole conversion site | Builder owns signature→bytes | No bypass path for `getFormattedSignature`; signature blob contract enforced |
| `_proto_utils.dart` shared module | Package-private library | Deduplicates proto3 varint logic between `passkey_barz_address.dart` and `barz_diamond_cut.dart` |
| `@visibleForTesting` challenge validator | Exposes `_validateChallenge` for pure-Dart tests | Enables full coverage of challenge mismatch path without FFI dependency |
| EIP-7702 rejects PasskeySigner | `ArgumentError` on `is!` check | 7702 authorization MUST be secp256k1; documented error message |
